CVE-2026-41992 · CVSS 7.5 · Jun 29, 2026 (updated Jul 01, 2026)

GNU gzip global buffer overflow in LZH decompression due to improper reuse of shared global state

GNU gzip contains a global buffer overflow vulnerability in the LZH decompression logic caused by improper reuse of shared global state between different decompression formats within a single execution. GNU gzip maintains a global array that is shared across the LZ77, LZW, and LZH decompression routines and is not reinitialized between files processed in the same invocation. By decompressing a specially crafted LZW file followed by a specially crafted LZH file in a single gzip -d command, an attacker can trigger the overflow.
