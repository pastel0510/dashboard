CVE-2026-41649 · Apr 28, 2026 (updated Apr 29, 2026)

Outline shares.create insecure direct object reference

The shares.create API endpoint starting in version 0.86.0 and prior to version 1.7.0 has an insecure direct object reference. When both collectionId and documentId are provided, the authorization logic only checks access to the collection, completely ignoring the document. This allows an authenticated attacker to generate a valid public share link for any document on the platform. Version 1.7.0 contains a patch.
