ChurchCRM SQL Injection via PropertyAssign.php

ChurchCRM is an open-source church management system. Prior to version 7.1.0, authenticated users with Edit Records or Manage Groups permissions can exploit a time-based blind SQL injection vulnerability in the PropertyAssign.php endpoint to exfiltrate or modify any database content, including user credentials, personal identifiable information (PII), and configuration secrets. The vulnerability has a CVSS v3.1 score of 8.1 (High) with Attack Vector: Network, Attack Complexity: Low, Privileges Required: Low, and User Interaction: None. CVE-2026-34986 · Apr 6, 2026

Go JOSE JWE Decryption Panic / Denial of Service

Go JOSE provides a Go implementation of JWS, JWE, and JWT standards. Prior to versions 4.1.4 and 3.0.5, decrypting a JSON Web Encryption (JWE) object will panic if the alg field indicates a key wrapping algorithm and the encrypted_key field is empty. The panic occurs in cipher.KeyUnwrap() when attempting to allocate a slice with a zero or negative length. The vulnerability has a CVSS v3.1 score of 7.5 (High) with Attack Vector: Network, Attack Complexity: Low, Privileges Required: None, and User Interaction: None. The impact is denial of service due to application panic. Upgrade to go-jose version 4.1.4 or 3.0.5 to patch this issue.
