CVE-2026-27130 · May 18, 2026

CVE-2026-27130 — Dokploy has Command Injection in its Service Operations

Dokploy is a free, self-hostable Platform as a Service (PaaS). Versions 0.26.6 and below have OS command injection through the appName parameter. Inadequate input sanitization, lack of schema validation, and direct shell interpolation allow an authenticated attacker to inject shell metacharacters (e.g., ;, $(), backticks, |, &) in the appName field during application creation. Injected commands are then executed with server-level privileges when service operations (start, stop, remove, scale) are triggered. This issue has been resolved in version 0.26.7. CVSS v3.1 score: 9.9 (Critical).

CVE-2026-7304 — SGLangs multimodal generation runtime is vulnerable to unauthenticated remote code execution when --enable-custom-logit-processor is enabled

SGLang multimodal generation runtime contains a flaw that permits execution of arbitrary code when the --enable-custom-logit-processor flag is activated. CVE-2026-7301 — SGLangs multimodal generation runtime scheduler ROUTER socket binds to 0.0.0.0 by default and calls pickle.loads() on incoming messages

SGLang multimodal generation runtime scheduler exposes a ROUTER socket that, by default, binds to 0.0.0.0 and directly deserializes incoming messages via pickle.loads(). CVE-2026-8507 · May 17, 2026

CVE-2026-8507 — Crypt::OpenSSL::PKCS12 for Perl has heap out-of-bounds write via large PKCS12 OCTET STRING attributes, enabling RCE

Crypt::OpenSSL::PKCS12 versions through 1.94 for Perl contain an out-of-bounds (OOB) heap write flaw. When parsing a PKCS12 file with a greater-than-or-equal 1 GiB OCTET STRING (or BIT STRING) attribute on a SAFEBAG via the info() or info_as_hash() functions, a signed integer overflow in the size calculation passed to Renew() triggers heap memory corruption with remote-code-execution potential. The vulnerability is fixed in version 1.95. CVSS v3.1 score: 9.8 (Critical).

CVE-2026-8721 · May 17, 2026

CVE-2026-8721 — Crypt::OpenSSL::PKCS12 for Perl truncates passwords containing embedded NULL bytes, reducing keystore security

Crypt::OpenSSL::PKCS12 versions through 1.94 for Perl truncate passwords with embedded NULL bytes because password parameters are declared as C strings. When strlen() is called on the buffer, bytes at or after the first NULL byte are silently discarded. Binary, KDF-derived, or HMAC-derived passwords lose entropy without any warning, weakening confidentiality of the PKCS12 keystore and facilitating key recovery attacks. The issue is fixed in version 1.95. CVSS v3.1 score: 9.8 (Critical).

CVE-2026-30950 — AutoGPT Authenticated Session Hijacking via IDOR in PATCH /sessions/{session_id}/assign-user

AutoGPT is a workflow automation platform for creating, deploying, and managing continuous artificial intelligence agents. Versions 0.6.36 through 0.6.50 are vulnerable to Authenticated Session Hijacking via IDOR. The PATCH /sessions/{session_id}/assign-user endpoint authenticates the caller but never verifies session ownership: the service layer invokes session lookup with user_id=None, which the data access layer interprets as a privileged/system call that bypasses the ownership filter, allowing any authenticated user to reassign an arbitrary session to themselves. The issue has been patched in version 0.6.51. CVSS v3.1 score: 7.1 (High).
