CVE-2026-27130 · May 18, 2026

Dokploy OS Command Injection via appName

[Critical, CVSS 9.9] Dokploy is a free, self-hostable Platform as a Service (PaaS). Versions 0.26.6 and below have OS command injection through the appName parameter. Three chained issues cause this problem: inadequate input sanitization, lack of schema validation, and direct shell interpolation. User-controlled application names are passed through inadequate sanitization (cleanAppName only replaces spaces and converts to lowercase) before being interpolated directly into shell commands executed via execAsync() and execAsyncRemote(). An authenticated attacker can inject shell metacharacters (e.g., ;, $(), backticks, |, &) in the appName field during application creation, executed with server-level privileges when service operations (start, stop, remove, scale) are triggered. Resolved in version 0.26.7.

CVE-2026-25244 · May 18, 2026

WebdriverIO Command Injection via Git Branch Name (RCE)

[Critical, CVSS 9.8] WebdriverIO is a test automation framework for unit, end-to-end, and component testing using WebDriver and Appium. Versions below 9.24.0 contain a command injection vulnerability leading to remote code execution (RCE) in test orchestration. Git permits branch names containing shell metacharacters, and getGitMetadataForAISelection() interpolates these names directly into execSync() calls without sanitization. An attacker exploits this by supplying a malicious repository whose branch name carries a payload, causing the shell to execute arbitrary code. This enables RCE on CI/CD servers and developer machines, leading to credential and secret disclosure, source code exfiltration, system compromise, and supply chain attacks via tampered build artifacts. Fixed in version 9.24.0.

lwIP snmpv3 USM Handler Stack-Based Buffer Overflow

[Critical, CVSS 9.8] A vulnerability was found in lwIP up to and including version 2.2.1. CVE-2026-26978 · May 18, 2026, updated May 19, 2026

FreePBX Backup Restore PHP Deserialization (RCE)

[High, CVSS 8.6] FreePBX is an open-source IP PBX platform. In versions below 16.0.71 and 17.0.6, the backup module does not properly sanitize data during restore operations. Selected files are extracted from a user-supplied tar archive and passed directly to PHP's unserialize() without validation, class restrictions, or integrity checks. A remote attacker with restoration privileges can exploit this to achieve arbitrary code execution on the web server (typically user asterisk or www-data) without shell access, CLI access, or write permissions beyond the normal restore workflow. Authentication with a known username having restore permissions is required. Fixed in v16.0.71 and v17.0.6.
