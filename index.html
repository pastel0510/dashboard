Flannel Kubernetes - Command Injection

The Flannel Extension backend is vulnerable to command injection via Node annotations. Attackers who can set Kubernetes Node annotations can achieve root-level arbitrary command execution on every flannel node in the cluster. The annotation content is unmarshalled and piped directly to a shell command without validation. Affects Flannel < v0.28.2. Use vxlan or wireguard backends as workaround.
