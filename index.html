CVE-2026-6073 · May 14, 2026

GitLab EE: Stored Cross-Site Scripting in Merge Request View

GitLab EE versions before 18.9.7, 18.10 before 18.10.6, and 18.11 before 18.11.3 contain a stored cross-site scripting (XSS) flaw. A maliciously crafted merge request is stored on the server and processed without proper output encoding. When a user who can view the merge request subsequently loads the page, the payload executes in the user's browser with the victim's session context. Because the payload is stored server-side, every user who views the affected merge request is a potential target. Exploitation does not require any cookies or special headers beyond the ability to read the affected page. The flaw is fixed in 18.9.7, 18.10.6, and 18.11.3. The vulnerability was assigned by GitLab (GitLab Issue #13) on 2026-05-14 and first seen in NVD on 2026-05-14.

CVE-2026-7377 · May 14, 2026

GitLab EE: Stored Cross-Site Scripting in Customizable Analytics Dashboards

GitLab EE versions before 18.9.7, 18.10 before 18.10.6, and 18.11 before 18.11.3 are affected by a stored cross-site scripting vulnerability in customizable analytics dashboards. Dashboard builders who introduce or edit widgets without sanitizing embedded values can plant a script payload into the dashboard definition. When another authenticated user subsequently views that dashboard, the stored payload is rendered into the page without proper output encoding and executes in the victim's browser session. The attack surface is an authenticated user who has the ability to create dashboards and controls the input that is later reflected to other users. Exploitation requires an authenticated user who can create or edit a dashboard; once the malicious dashboard is published, every viewer is an automatic target. The flaw was reported by GitLab on 2026-05-14 and assigned that same day. It remains confirmed not in the CISA KEV catalog.

CVE-2026-8657 · May 16, 2026

jsondiffpatch: Prototype Pollution via Unfiltered Delta or JSON Patch Input

Versions of the package jsondiffpatch before 0.7.6 are vulnerable to prototype pollution via the jsondiffpatch.patch() and jsondiffpatch/formatters/jsonpatch.patch() APIs. These functions accept a delta or JSON Patch document from a caller and use attacker-controlled property names and path segments to traverse and mutate arbitrary object trees. Because the traversal does not guard against special properties such as __proto__ or constructor.prototype, an attacker who controls the patch document can write to Object.prototype, causing every subsequently constructed JavaScript object to inherit the poisoned property. The consequence is reliable remote code equivalent: a payload can modify prototypes used by downstream code — for example, JSON.parse — which an attacker can chain into code execution. Any application accepting or applying JSON Patch or jsondiffpatch-style diffs from an untrusted input vector is at risk. The vulnerability was reported to Snyk, who reserved CVE-2026-8657 on 2026-05-15 and published it on 2026-05-16. Scores shown are maximum CVSS v4.0 across NVD versions.
