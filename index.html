CVE-2026-31222 · May 12, 2026 · May 12, 2026

Snorkel library insecure deserialization via Trainer.load()

The snorkel library through v0.10.0 contains an insecure deserialization vulnerability (CWE-502) in the Trainer.load() method. The method loads model checkpoint files using torch.load() without weights_only=True, allowing arbitrary Python object deserialization via Pickle. A remote attacker can supply a malicious model file to execute arbitrary code on the victim system.

Snorkel library insecure deserialization via BaseLabeler.load()

The snorkel library through v0.10.0 contains a critical insecure deserialization vulnerability (CWE-502) in the BaseLabeler.load() method. The method loads serialized labeler models using unsafe pickle.load() on user-supplied file paths without validation. CVE-2026-31225 · May 12, 2026 · May 12, 2026

superduper project remote code execution via eval() in query parsing

The superduper project through v0.10.0 contains a critical remote code execution vulnerability in its query parsing component. The _parse_op_part() function in query.py uses unsafe eval() to dynamically evaluate user-supplied query operands without proper sanitization. Although a restricted global namespace is provided, dangerous built-in functions are not blocked, allowing an attacker to import modules (e.g., os) and execute arbitrary system commands.

CVE-2026-31242 · May 12, 2026 · May 12, 2026

mem0 server unauthenticated database reset via DELETE /memories

The mem0 v1.0.0 server lacks authentication and authorization controls for its memory reset functionality accessible via the DELETE /memories endpoint. An unauthenticated attacker can send a DELETE request that triggers a reset operation, executing a DROP TABLE SQL statement and deleting the entire memory database table, causing catastrophic data loss and complete denial of service for all users.
